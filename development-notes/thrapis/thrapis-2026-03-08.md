- **Описание происходящего:** Ладно, самое время заняться добавлением странички решения задачи с интеграцие ИИ (для суждения првильности решения). Кстати, девочки, 8 марта же, с праздником!
- **Проблемы:** API для фронта сделал, но там не хватает суждении ИИ по решению. Очевидно, что даже базовой страницы задачи - нет.
- **Решения (или попытки):** Сначала нужно было придумать, каким способом сстраница понимала, какую задачу нужно дать на решения. Тут придумал! Пускай у страницы будут query-параметры с идентификаторами категории задачи и её сложности - там уже пускай запрашивает случайную задачу по этим параметрам (ну, не идентификатор же задачи передавать со страницей - это было бы странно). Также пусть следит за изменением значений этих параметров через watch (у нас тут сайт без полных перезагруок страниц, так необходимо для плавной работы).

  Ладно, вернёмся к бэку. Добавим новые параметры окружения: ключ для ИИ-сервиса, endpoint для обращения к ИИ и название самой модели ИИ. Добавим необходимые DTO для общения с сервисом. Теперь в API backend-а можно интегрировать обращение к ИИ-сервису, используя опыт, набранный при тестировании взаимодействия в проекте с Golang. Хм, также, наверное, нужно оптимизировать свою старую инструкцию проверки, чтобы ИИ не отклонялся от идеии проверки (странные ответы, попытки манипуляции от пользователя).

  Теперь можно дальше оформить страничку решения задачи. Описания задачек и ответы от ИИ формируются в формате Markdown, потому для такого я нашёл для Nuxt модуль MDC, который и позволяет визуализировать md-формат. Конечно, тут пришлось поборотся с конфликтами Tailwind, но вроде всё в порядке. Применил давече установленнный модуль Icons, чтобы на странице не только "букавы читать". Гифка загрузки для лучшего UX. Кнопка поиска новой задачи начинает заново цикл поиска задачи и его решения пользователем.

- **Мысли / Планы:** Стоит заняться решением вопроса с повышением уровня пользователя, если он добрал достаточно очков опыта.
